Hypermedia TODO App

The Problem:

A problem with modern web development is that people often reach directly for JavaScript frameworks, like Angular and React to create their web apps rather than considering if a hypermedia application would be a better way to build their app. The purpose of this project was to show how a hypermedia approach can be used to create a competent and fully functional web app with a quality user experience.

What tools did we use?

Front End

Html:

Since we are using a hypermedia approach our application was formatted in HTML

Pico CSS:

We used the css library pico css to give us a functional and asthetic layout. We did in a few small intstances write some in-line styles to style elements to our needs

HTMX:

HTMX is a JavaScript library that allows more hypermedia controls to be injected into html. It gave us as developers the power to add lots of functionality and quality of life improvements without large JavaScript frameworks.

JavaScript

Although the point of this project was to show how a hypermedia approach can be used to build powerful web apps there were some instances where we used snippets of JavaScript to add some functionality.

Back End

Python:

We used python as the language that all of our backend code is written in.

Flask:

We used the python library Flask to be our webserver, this was my second time using Flask and I feel that it is one of the best approches to get a webserver up and running in python. It is really intuitive and easy to use by handling your routing for you.

Jinja2

Jinja is an HTML templating language that feels like an obvious option for our needs in this project it perfectly bridges the gap between HTML and python. By giving us the ability to have dynamically generated HTML our apps functionality and user experience was greatly improved.

PeeWee ORM with SQLite:

PeeWee is an object relational model that allowed us to connect our backend to a SQLite database. Its syntax was easy to learn and doesn't feel too far off from writing SQL directly in python but it gives you a few helpful guardrails.

Our Development Process:

The first portion of our development was done together as a class. After that portion of the development was done we split off into groups and spent time brainstorming features that would improve the user experience. We then seperatly implemented features that improved the user experience by providing more features and security.

Web 1.0:

The first iteration of our application was developed with a web 1.0 mindset. That meant most of our interaction with the user was occuring via forms issuing post and get requests. We began by outlining what our app was going to need to do. We took this and built the intial layout and functionality. After that was completed we implemented a connection to our database via PeeWee, this required us to reimplement the way our data model was designed but the trade-off was definitly worth it. The ability to have our TODOs persist past memory was highly valuable by speeding up our development process.

Adding HTMX

The addition of HTMX to our application added lots of small quality of life changes that really made the user experience more smooth. These changes allowed the application to update the page while maintaining scroll state. This included toggling the completness of TODOs, reorder of TODOs and active search that updated results as a user provided input.

Expanding Functionality

My group had three members and we each decided on a feature or an area of the user experience that we wanted to improve. We decided on application security, adding the ability to have multiple users, and adding information onto a users TODO.

For security implemented some HTML escaping to prevent XSS(Cross-Site Scripting) attacks. This user input sanitization can give users a better piece of mind that their information is secure within our system. We also implemented feedback to the user, if a user tries to bypass the required attribute in the HTML and enter an invalid TODO with missing information they are given a notification window indicating what error occured.

The addition of multiple users in our system may seem obvious but it was important that our first iteration was functioning how we intended before we expanded to adding this feature. This feature allows our application to be much more powerful. A user is able to create a username and password which they can use on their subsequent visits to our site to access their TODOs. This change required some changes to our database by giving a todo a userID which is associated with a user and filter all TODOs based on that ID.

Since we had spent some time defining our goals we had a clear vision on what user experience improvements needed to be made.
